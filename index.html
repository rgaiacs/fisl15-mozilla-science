Early First Year Birthday

Mozilla has a long history of push forward the web with many projects like WebFWD, PDF.js, TogetherJS, OpenBadges, Persona, Popcorn, Firefox OS, WebMaker, ...

In June 14, 2013, Mozilla start pushing forward the science in the web with Science Lab

Help researchers use the power of the open web to change science's future

This is the mission of Mozilla Science Lab.

What is open science?

Not just gratis PDF article!!!

  • access to content, data, code, materials, ...
  • emergence of web-native tools
  • rewards for openness, interoperability, collaboration, sharing
  • push for ROI (Return on Investment), recomputability, transparency
Open science is a umbrela term that cover many of new science/research practices.

Teach

The curriculum's are already full. Yet they still rooted in 17th c. practices.
  • unix shell
  • control version (e.g. Git and GitHub)
  • good programming practices with Python or R
  • databases
Two days bootcamps that teach introduce this concepts to the audience.

Workshops

Around 80 workshops in the first year.

Rewards

Not just papers citations.

Code as a research object

Start give the same importance of articles to (a) softwares and (b) data providing a Digital Object Identifier.

Prototype and more information at http://mozillascience.github.io/code-research-object/.

Two days bootcamps that teach introduce this concepts to the audience.

Code Review in Science

An important part of foss development is code review. Unfortunately just a small amount of scientific source code are review.

First Study

  • Ran in second semester of 2013 and have PLOS as partnership.
  • 11 developers (most from Mozilla) and 4 authors of PLOS papers.
  • The scientists appreciated the review.
  • The developers felt constrained by their lack of domain knowledge.

More informations at this arXiv pre-print.

Second Study

  • Running right now (12-weeks long).
  • More like coaching by "experts" developers.
